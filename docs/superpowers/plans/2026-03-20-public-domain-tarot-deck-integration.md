# Public Domain Tarot Deck Integration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a complete locally hosted public-domain Rider-Waite-Smith tarot deck to the product, expand the runtime draw deck to 78 cards, and render tarot card images on the reading result page.

**Architecture:** Introduce a canonical 78-card deck definition for runtime draws, add a repeatable downloader that fetches and normalizes the approved Wikimedia Commons card files, and generate a manifest that the web app can use to resolve `card_id` to local image assets. Keep API contracts unchanged and confine UI changes to the reading result presentation layer.

**Tech Stack:** NestJS, Next.js, TypeScript, tsx, Node.js fetch/fs, Jest
